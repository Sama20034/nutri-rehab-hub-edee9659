

## المشكلة

صفحة تسجيل الدخول (`Auth.tsx`) بتعمل redirect تلقائي لما يكون في session موجود. Supabase بيستخدم `localStorage` فكل التابات بتشارك نفس الـ session. لما الأدمن يسجل دخول، أي تاب جديد بيتحول للداشبورد مباشرة ومفيش طريقة لتسجيل الخروج أو الدخول بحساب تاني.

## الحل

تعديل صفحة `Auth.tsx` بحيث لما يكون في مستخدم مسجل دخول بالفعل، بدل ما يتعمل redirect تلقائي، نعرض شاشة فيها خيارين:

1. **"متابعة كـ [اسم المستخدم]"** — يروح للداشبورد المناسب
2. **"تسجيل دخول بحساب آخر"** — يعمل `signOut()` ويعرض فورم تسجيل الدخول

### التغييرات المطلوبة

**ملف واحد: `src/pages/Auth.tsx`**

- إزالة الـ `useEffect` اللي بيعمل auto-redirect (سطر 38-56)
- إضافة state جديد `switchingAccount` 
- لما يكون `user` موجود و `switchingAccount === false`: عرض كارد فيه:
  - رسالة "أنت مسجل دخول كـ [email]"
  - زرار "متابعة للداشبورد" → يعمل navigate للداشبورد المناسب
  - زرار "تسجيل دخول بحساب آخر" → يعمل `signOut()` ويخلي `switchingAccount = true`
- لما `switchingAccount === true` أو مفيش user: يعرض فورم تسجيل الدخول العادي

مفيش تعديلات على الـ database أو backend.
