

# إلزام تسجيل الدخول قبل الشراء

## المشكلة
حالياً أي زائر يقدر يشتري كـ guest بدون تسجيل، وبالتالي مش هيقدر يتابع طلباته بعد كده.

## الحل
تعديلات بسيطة في مكانين:

### 1. صفحة Checkout (`src/pages/Checkout.tsx`)
- إضافة `useEffect` يعمل redirect للمستخدم غير المسجل لصفحة `/auth` مع رسالة توضيحية.
- إزالة حقول الـ guest (الاسم والإيميل) من الفورم لأنها مش هتبقى مطلوبة.
- تبسيط logic الطلب لأن كل الطلبات هتكون مرتبطة بـ `user.id`.

### 2. صفحة المتجر (`src/pages/Store.tsx`)
- زر "Checkout" في السلة: لو المستخدم مش مسجل، يوديه لصفحة `/auth` بدل `/checkout` مع toast يقوله "سجل الأول".

### 3. إضافة صفحة "طلباتي" (اختياري - مرحلة تانية)
- ممكن نضيفها بعدين في الـ Client Dashboard أو كصفحة مستقلة لمتابعة الطلبات.

## الملفات المتأثرة

| ملف | تعديل |
|------|--------|
| `src/pages/Checkout.tsx` | redirect لغير المسجلين + إزالة guest fields |
| `src/pages/Store.tsx` | زر Checkout يطلب تسجيل الدخول أولاً |
| `src/hooks/useCart.tsx` | إبقاء guest cart كما هو (السلة تشتغل بدون تسجيل، بس الشراء يحتاج تسجيل) |
